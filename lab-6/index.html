Lab 6 Addendum

In this lab I implemented a classic ‚ÄúHoly Grail‚Äù web layout using CSS Grid. The layout has five regions: header, navigation sidebar, main content, an advertisement sidebar, and a footer. The grid template defines columns for nav, content, and ad, and keeps header/footer spanning full width.

For the JavaScript portion, I built an interactive Bubble Sort tool. The user types a list of comma-separated numbers and clicks ‚ÄúSort Numbers.‚Äù The script (in an external script.js file) parses the input into an array, then runs a Bubble Sort loop. It records each comparison/swap step, counts total swaps, and displays: (1) the original array, (2) the sorted array, (3) stats about how many swaps happened, (4) an optional step-by-step trace. This satisfies the requirement that the page responds to an event (button click).

Challenges: Getting clean user input was tricky. Users might type spaces or double commas, which can produce NaN. I solved this by trimming each value and ignoring blanks before converting to Number. Another challenge was keeping JavaScript in a separate file while still updating elements in the DOM. I handled that by giving key elements unique id attributes (like sortedOutput) and using document.getElementById(...) in script.js.
